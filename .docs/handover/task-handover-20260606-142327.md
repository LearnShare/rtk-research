## 研究探索流程

### 方法论

本研究采用 **"广度优先 → 纵深突破 → 实操验证 → 安全审计"** 的四层递进模式，分两期执行：

```
v1：广度覆盖（7 阶段 28 交付物）
  ├── P0 环境搭建 → 编译、命令实测、Hook 验证
  ├── P1 架构理解 → 路由、模块、Core、过滤、测试
  ├── P2 机制深挖 → 过滤引擎、Hook、追踪、TOML DSL、Trust
  ├── P3 质量评估 → 测试体系、集成测试、性能基准
  ├── P4 生态分析 → 演进、社区、外部文章
  ├── P5 对比研判 → 工具对比、技术趋势
  └── P6 知识整合 → 报告、知识图谱、贡献指南

v2：纵深突破（6 阶段 17+ 交付物）
  ├── Q0 实操突破 → learn/parser/stream/discover 深挖 + 2 个 filter 实践
  ├── Q1 安全审计 → 命令注入、沙箱、完整性、供应链
  ├── Q2 性能剖析 → hyperfine 基准、依赖分析
  ├── Q3 未探模块 → runner/ccusage/hooks 剩余
  ├── Q4 纵深研究 → 版本差异、跨平台、错误模式
  └── Q5 知识缝合 → 技术债务、白皮书、二次开发方案
```

### 核心工作原则

1. **每任务独立交付**：每个任务完成后立即保存交付物，不积压
2. **状态持续跟踪**：研究计划标记任务状态（⬜/🏗️/✅），TaskCreate 跟踪执行
3. **交付物模板化**：研究日期 + 阶段 + 方法 + 概述 + 核心发现 + 详细分析 + 结论
4. **真实数据优先**：运行真实命令、捕获真实输出、检查真实测试结果
5. **双路径验证**：已安装二进制（v0.37.2）实测 + 源码（v0.42.2）分析

## 关键发现 Top 10

1. **双轨过滤引擎**：Rust-native（状态机/JSON 解析）+ TOML DSL（8 阶段声明式管道），覆盖 100+ 命令
2. **Hook 4 层安全**：白名单 → SHA-256 完整性 → 权限决策（Deny/Allow/Ask）→ 项目信任管理
3. **`rtk rewrite` exit code 契约**：0=允许 / 1=无匹配 / 2=拒绝 / 3=需确认，Hook 重写单一真相源
4. **Token 追踪架构**：SQLite bundled + RAII TimedExecution + 项目级 GLOB + 90 天轮换
5. **2065 测试全绿**：insta 快照 + ≥60% Token 硬约束 + TOML `[[tests.*]]` 双重验证
6. **74 条重写规则**：覆盖 12+ 类别，复合命令 `&&`/`|`/`;` 感知，不可审核构造检测
7. **4 个月 40+ 版本**：v0.2.1(1月)→v0.42.2(6月)，20+ 贡献者，1415 提交
8. **13 种 AI Agent 集成**：Claude/Cursor/Copilot/Gemini/Cline/Windsurf/Kilo Code/Codex CLI/OpenCode/OpenClaw/Pi/Hermes/Antigravity
9. **二级编译保障**：build.rs 构建时验证 TOML + 检测重复 filter 名称，编译失败阻止问题进入二进制
10. **`unsafe_code = "deny"` 严格执行**：仅 2 处 Unix 信号处理使用 unsafe，其余全 safe Rust
