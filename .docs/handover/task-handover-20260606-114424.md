## 研究和执行成果

### 核心发现 Top 5

1. **双轨过滤引擎**：Rust-native（状态机/JSON 解析）处理复杂结构化输出，TOML DSL（8 阶段声明式管道）覆盖 60+ 长尾工具
2. **Hook 4 层安全模型**：操作命令白名单 → SHA-256 完整性校验 → 权限决策（Deny/Allow/Ask）→ 项目 TOML 信任管理
3. **`rtk rewrite` exit code 契约**：0=允许 / 1=无匹配 / 2=拒绝 / 3=需确认，构成 Hook 重写单一真相源
4. **Token 追踪架构**：SQLite bundled + RAII `TimedExecution` + 项目级 GLOB 匹配 + 90 天自动轮换
5. **2065 测试全绿**：快照测试（insta）+ ≥60% Token 节省硬约束 + TOML 内联测试 `[[tests.*]]` 双重验证

## 错误和反思

### 已解决

1. **Rust 未安装** → 尝试 3 种安装方式后确认已安装，跳过流程
2. **自动安全策略阻止操作** → 被 `auto mode classifier` 拦截多次（file 命令、网络测试、安装程序执行），最终通过用户调整权限模式为 `dontAsk` 解决
3. **代理网络下载** → 大陆网络限制 rustup 下载，配置 `127.0.0.1:10808` 代理解决
4. **`rtk grep --path` 失败** → v0.37.2 不支持该参数，以 v0.42.2 源码分析为准
5. **WebFetch 联网被阻止** → 外部文章以搜索摘要代替

### 待改进

1. **v0.37.2 vs v0.42.2 版本差异**：已安装二进制和源码版本不一致，部分功能测试受限
2. **集成测试跳过**：`cargo test --ignored` 的 7 个测试需要 `cargo install --path .` 更新到 v0.42.2
3. **WebFetch 受限**：外部文章未能完整阅读，信息可能不完整
4. **hyperfine 未安装**：性能测试仅做了基础测量，未做系统基准
5. **仅 Windows 环境**：跨平台行为（Unix SIGPIPE、macOS Homebrew）未验证
