# build-verification — RTK 编译与安装环境验证

> **研究日期**：2026-06-06
> **研究阶段**：P0.1
> **研究方法**：实际操作

## 概述
验证 RTK 开发工具链和已安装二进制的工作状态。

## 核心发现

1. **RTK 已安装**：v0.37.2，二进制 8.17 MB，安装在 `/c/works/rsync/rtk`
2. **Rust 工具链**：v1.96.0（满足 MSRV ≥ 1.91）
3. **版本差异**：已安装 v0.37.2 vs 源码 v0.42.2（源码更新但未重新编译）
4. **`rtk gain` 正常**：已追踪 139 条命令，累计节省 21.7K tokens (10.5%)
5. **RTK hook 未激活**：当前使用 `CLAUDE.md @RTK.md` 方式而非 PreToolUse hook

## 详细分析

### 环境状态

| 项目 | 值 | 备注 |
|------|-----|------|
| 操作系统 | Windows 11 (MSVC) | x86_64-pc-windows-msvc |
| RTK 版本 | v0.37.2 | 已安装二进制 |
| RTK 源码版本 | v0.42.2 | 仓库 develop 分支 |
| Rust 版本 | 1.96.0 | MSRV 1.91，满足要求 |
| 二进制路径 | `/c/works/rsync/rtk` | 非标准安装位置 |
| 二进制体积 | 8,565,248 字节 (~8.17 MB) | 小于 10MB |
| 配置目录 | `~/.config/rtk/` | Unix 风格路径 |

### 工作原理
当前 RTK 通过 `CLAUDE.md` 中的 `@RTK.md` 指令工作：Claude Code 在每次 Bash 命令前自动插入 `rtk` 前缀，实现命令重写。这种方式的覆盖率为 100%（所有 Bash 命令），但不如 PreToolUse hook 透明。

## 结论与洞见

- 环境可用，无需重新编译即可进行源码研究和功能验证
- 版本差异（0.37.2 vs 0.42.2）需要在后续分析中注意，部分新功能可能在源码中存在但本地二进制不支持
- 后续 P1-P6 源码分析将以 v0.42.2 源码为准
